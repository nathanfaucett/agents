## Summary
This skill takes an Agents Loop markdown file that includes explicit sub-agent checkpoint workflow and executes the next pending task. It updates the file with progress and signals for the next sub-agent to pick up work in sequence. This enables iterative execution of plans generated by the `subagents-tasks` skill, allowing for a structured workflow across multiple sub-agents. Each task should be executed in a subagent with a fresh context, and the markdown file should be updated to reflect the current state of the workflow, including any blockers, next steps, and signals for the next sub-agent.

## Step by step process
1. Read the specified Agents Loop markdown file from the filesystem.
2. Parse the file to identify the next pending task in the `Sub-agent workflow` section.
3. Execute the identified task in a sub-agent with a fresh context.
4. Update the markdown file to mark the task as `In progress` and include any relevant details about the execution (e.g., outputs, blockers, next steps).
5. Once the task is completed, update the markdown file to mark the task as `Done` and include any relevant outputs or results.
6. Add signals for the next sub-agent to pick up the next pending task, and update the `Next steps` section accordingly.
7. Write the updated markdown file back to the filesystem, ensuring it remains sub-agent checkpoint friendly for the next execution cycle.
